# Repository Governance

## Source hierarchy
1. Verified official/legal evidence.
2. Approved controlled project documents.
3. Primary research and third-party commercial/technical evidence.
4. Working assumptions explicitly labelled as such.
5. Generated concepts and exploratory visuals.

Lower-ranked material must never silently override higher-ranked evidence.

## Document states
`DRAFT` → `REVIEW` → `APPROVED` → `ISSUED` → `SUPERSEDED`.

## Naming convention
`OBL-[DOMAIN]-[DOC]-[SHORT-TITLE]-vMAJOR.MINOR-YYYYMMDD-[STATUS].[ext]`

## External release gate
External material must have factual, claims, financial and confidentiality review appropriate to its audience.

## Security
Do not commit personal data, credentials, private keys, bank information, confidential supplier quotations, unredacted corporate records, raw consumer data or privileged legal material to a public repository.
